# 第3章：张量与神经网络基础

## 📚 本章学习目标

1. 理解张量的概念和核心属性（shape），能将移动端开发中的数组、矩阵等数据结构与张量对应起来
2. 掌握神经网络的本质——函数拟合，理解线性变换 + 激活函数如何构建非线性模型
3. 理解前向传播、损失函数、反向传播和梯度下降的完整训练流程
4. 能够使用 PyTorch 的 `nn.Module` 和优化器实现标准训练循环（前向 → 损失 → 反向 → 更新 → 清零）
5. 通过可视化直观感受损失下降、参数收敛和学习率对训练的影响

## 🎯 核心知识点

1. **张量（Tensor）**：任意维度的数组容器。0维=标量，1维=向量，2维=矩阵，3维及以上=高维张量。大模型的输入通常是 3 维张量 `(batch_size, seq_len, hidden_dim)`，shape 是张量最重要的属性
2. **神经网络的本质**：一个从数据中自动学习规则的函数。核心操作是线性变换（矩阵乘法 `y = Wx + b`）加激活函数（引入非线性），层数越深能拟合越复杂的模式
3. **前向传播**：数据从输入层到输出层的单方向流动，每一层执行矩阵乘法 + 加偏置 + 激活函数，类似 iOS 的 View 渲染管线
4. **损失函数**：衡量预测值与真实值的差距。回归任务用 MSE（均方误差），分类和语言模型用交叉熵（Cross Entropy）。损失越小模型越好
5. **反向传播与梯度**：从损失出发，利用链式法则反向计算每个参数的梯度（调整方向和幅度）。PyTorch 用 `loss.backward()` 自动完成
6. **梯度下降**：参数更新公式 `w = w - lr * gradient`。学习率控制步长——太大会震荡发散，太小收敛过慢
7. **训练循环五步法**：前向传播 → 计算损失 → 反向传播 → 参数更新 → 梯度清零，这是深度学习训练的标准模式

## ❓ 常见问题 FAQ

**Q1: 张量和 NumPy 数组有什么区别？为什么不直接用 NumPy？**

PyTorch 张量和 NumPy 数组在功能上类似，但张量有两个关键优势：(1) 可以在 GPU 上运算，获得数十倍的加速；(2) 支持自动求导（autograd），能自动计算梯度用于训练。两者可以通过 `torch.from_numpy()` 和 `.numpy()` 方便地互转，但在训练流程中必须使用张量。

**Q2: 为什么每次 `loss.backward()` 之前必须调用 `optimizer.zero_grad()`？**

因为 PyTorch 的梯度默认是累加的，不是覆盖。如果不清零，每次 `backward()` 计算的梯度会叠加到之前的梯度上，导致参数更新方向错误。这是初学者最常见的 bug。记住口诀：清零 → 反向 → 更新。

**Q3: 激活函数为什么这么重要？没有激活函数会怎样？**

如果没有激活函数，无论堆叠多少层线性层，整个网络等价于一个线性层（矩阵乘法的结合律）。激活函数引入非线性，让网络能拟合曲线、波形等复杂模式。ReLU 是最常用的通用选择，GELU 是 Transformer 模型（如 GPT、BERT）的标配。

**Q4: 学习率应该设多大？有没有经验值？**

没有万能答案，但有经验范围：简单线性回归用 0.01-0.1，复杂深层网络用 1e-3 到 1e-4，预训练大模型用 1e-4 到 3e-4，微调大模型用 1e-5 到 5e-5。实践中建议先用默认值（如 Adam 优化器的 1e-3），观察损失曲线，再根据收敛情况调整。

**Q5: practice.ipynb 中的 MLP 拟合 sin(x) 用了 1000 个 epoch，大模型训练也需要这么多轮吗？**

大模型训练通常只需要 1-3 个 epoch，因为训练数据量极大（数百 GB 文本），一个 epoch 就包含了数十亿个 token。而我们的小实验数据量很少（几百个点），所以需要多轮重复学习。Epoch 数量和数据量是此消彼长的关系。
