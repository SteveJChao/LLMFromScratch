# 第4章：词嵌入（Embedding）的奥秘

## 📚 本章学习目标

1. 理解为什么需要把文字变成数字，以及什么样的数字表示才是"好"的——能反映语义关系
2. 掌握 One-hot 编码的原理及其三大致命问题（维度爆炸、无法表达相似性、无法泛化）
3. 理解词嵌入（Word Embedding）的核心思想：用稠密低维向量表示词，让语义相近的词在向量空间中靠在一起
4. 理解 Word2Vec 的训练原理（CBOW 和 Skip-gram），以及"国王 - 男人 + 女人 ≈ 女王"背后的几何含义
5. 掌握 PyTorch `nn.Embedding` 的使用方法，理解 Embedding 层在大模型中作为第一层的角色

## 🎯 核心知识点

1. **文字转数字的必要性**：神经网络只能处理数字，必须将文字转为数值表示。好的编码应让语义相近的词在数值上也相近，就像给城市标经纬度而非随意编号
2. **One-hot 编码**：每个词用一个只有一个 1 其余全是 0 的向量表示。三大问题：维度等于词表大小（5万+维）极度稀疏；任意两个不同词的相似度都是 0；模型无法利用词与词之间的语义关系进行泛化
3. **词嵌入（Word Embedding）**：用稠密、低维（如 256 维）的向量代替稀疏、高维的 One-hot 向量，每个维度都携带语义信息。核心假设——出现在相似上下文中的词含义相似（分布式假设）
4. **Word2Vec**：经典的词嵌入训练方法。CBOW 用上下文预测中心词（完形填空），Skip-gram 用中心词预测上下文（联想周围）。训练过程与标准神经网络一致：前向→损失→反向→更新
5. **词向量的几何魔法**：词向量支持语义加减法，如"国王 - 男人 + 女人 ≈ 女王"。不同语义关系（性别、国家-首都、时态）对应向量空间中不同的平行方向
6. **余弦相似度**：衡量两个词向量方向一致性的指标，值域 [-1, 1]。1 表示语义极其相似，0 表示语义无关，-1 表示方向完全相反
7. **nn.Embedding 层**：本质是一个可训练的查表操作，根据词 ID 直接查出对应向量。数学上等价于 One-hot 乘以权重矩阵，但查表效率远高于矩阵乘法。矩阵 shape 为 (vocab_size, embed_dim)，是大模型的第一层

## ❓ 常见问题 FAQ

**Q1: 词嵌入和 One-hot 编码的本质区别是什么？**

One-hot 编码是稀疏的、高维的（维度等于词表大小），且任意两个不同词的相似度都是 0，无法反映语义关系。词嵌入是稠密的、低维的（通常 64~768 维），每个维度都携带语义信息，语义相近的词向量也相近。One-hot 不需要训练但没有语义，词嵌入需要从数据中学习但能编码丰富的语义关系。

**Q2: 为什么"国王 - 男人 + 女人 ≈ 女王"能成立？**

因为词向量空间中存在规律的几何结构。"国王"和"男人"的差向量编码了"皇室"这个语义方向，将这个方向加到"女人"上就得到了"女王"。本质上，性别关系在词向量空间中形成了平行的方向，所以向量加减法在语义上也是合理的。这不是人为设计的，而是模型从大量文本的上下文共现关系中自动学到的。

**Q3: nn.Embedding 和普通的矩阵乘法有什么区别？**

数学上 `nn.Embedding(词ID)` 等价于 `One-hot向量 @ 权重矩阵`，结果完全相同。但 Embedding 通过索引直接取矩阵的某一行（O(1) 操作），而矩阵乘法需要做 vocab_size x embed_dim 次乘加运算。当词表有 5 万个词时，Embedding 比矩阵乘法快上万倍。

**Q4: 预训练词向量（如 Word2Vec、GloVe）和大模型中的 Embedding 层有什么关系？**

预训练词向量是在大规模语料上独立训练的静态词表示，每个词只有一个固定向量。大模型（如 GPT、BERT）中的 Embedding 层是模型的一部分，其权重会在整个模型训练过程中一起更新。早期的做法是用预训练词向量初始化 Embedding 层，但现代大模型通常从随机初始化开始训练，因为后续的 Transformer 层能学到更丰富的上下文相关表示。

**Q5: 词嵌入的维度（如 256、768）是怎么确定的？**

没有严格的理论公式，主要靠经验和实验。一般规律是：词表越大、任务越复杂，需要的嵌入维度越高。常见的选择有 Word2Vec 用 100~300 维，BERT-base 用 768 维，GPT-3 用 12288 维。维度太小会丢失语义信息，太大会增加计算量且可能过拟合。实践中通常选择 2 的幂次（64、128、256、512、768）便于 GPU 并行计算。
