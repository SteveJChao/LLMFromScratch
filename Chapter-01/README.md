## 🎯 核心知识点

1. **RNN -> LSTM -> Attention -> Transformer**：核心升级是从串行处理到并行处理，从局部记忆到全局注意力
2. **Attention 机制**：让每个词都能直接"看到"所有其他词，解决了长距离依赖和并行计算两大问题
3. **GPT = 生成（Decoder-Only），BERT = 理解（Encoder-Only），T5 = 转换（Encoder-Decoder）**：GPT 架构因生成能力强而成为当前主流
4. **Scaling Law**：模型性能与参数量、数据量、计算量呈可预测的幂律关系，三者需同步增长
5. **大模型训练三阶段**：预训练（学知识）-> 监督微调 SFT（学指令）-> RLHF（学偏好）
6. **"预测下一个词"的深层含义**：为了准确预测下一个词，模型被迫学会语法、常识、推理等所有能力
7. **解码策略**：贪心解码确定但易重复，采样（Temperature / Top-k / Top-p）在多样性和质量之间权衡

## ❓ 常见问题 FAQ

**Q1: Transformer 和 Attention 是什么关系？**

Attention（注意力机制）是一种计算方法，让每个词能直接关注到所有其他词。Transformer 是基于 Attention 构建的完整模型架构，包含多层 Attention + 前馈网络（FFN）+ 残差连接 + Layer Normalization 等组件。简单说，Attention 是核心零件，Transformer 是完整的机器。

**Q2: 为什么 GPT 只能从左往右看，不能像 BERT 那样看两边？**

因为 GPT 的任务是"生成下一个词"。在生成时，后面的词还不存在，所以训练时也只能看左边的内容（因果掩码）。BERT 的任务是"完形填空"，前后文都已知，所以可以双向看。这个设计差异决定了 GPT 擅长生成，BERT 擅长理解。

**Q3: 大模型的"涌现能力"是怎么回事？为什么模型大了就突然"开窍"？**

涌现能力是指模型在达到某个参数量临界点后，突然表现出之前完全不具备的能力（如数学推理、代码生成）。目前学界对此的解释尚未完全统一，但主流观点认为：当模型足够大时，内部形成了更复杂的知识表示和推理路径，使得这些能力从量变到质变。

**Q4: 为什么 practice.ipynb 中同一个提示词，采样多次会得到不同的结果？**

因为采样（sampling）是按概率分布随机选择 token，每次随机选择的结果可能不同。Temperature 参数控制随机程度：温度越高越随机（更有创意但可能不连贯），温度越低越确定（更保守但可能重复）。贪心解码（Greedy）每次都选概率最高的 token，所以结果是确定的。

**Q5: GPT-2 只有 124M 参数，现在的模型动辄几百B，差距在哪里？**

参数量的增加带来：(1) 更多的知识存储容量，(2) 更强的推理能力，(3) 更好的指令遵循能力。但核心架构（Transformer）是一样的，区别主要在层数、隐藏维度、注意力头数的增加，以及训练数据量和训练技巧的提升。本课程从小模型入手，逐步理解大模型的每个组件。
