# Chapter 07: 位置编码——让模型理解顺序

> **预计学习时间：约 30 分钟**

## 学习目标

1. 理解为什么注意力机制需要位置编码（置换不变性问题）
2. 掌握正弦位置编码的原理和实现（不同频率的 sin/cos 编码不同尺度的位置信息）
3. 理解可学习位置编码的工作方式（nn.Embedding 查找表）
4. 了解 RoPE 旋转位置编码的核心思想（通过旋转 Q/K 编码相对位置）
5. 能通过实验验证位置编码对注意力模式的影响

## 核心知识点

1. **置换不变性** — 注意力机制本身不区分词的先后顺序，打乱输入序列不影响每个词的注意力输出
2. **位置编码的本质** — 给每个位置一个独特的向量，加到词嵌入上，让模型能区分相同词在不同位置的含义
3. **正弦位置编码** — 用不同频率的 sin/cos 函数编码位置，低维度高频（区分相邻位置），高维度低频（区分远距离），值域有界且可泛化到任意长度
4. **可学习位置编码** — nn.Embedding(max_len, d_model) 查找表，更灵活但有最大长度限制，GPT-2/BERT 使用
5. **RoPE 旋转位置编码** — 通过旋转 Q/K 向量来编码位置，点积只取决于相对位置差，被 LLaMA、DeepSeek、Qwen 等现代大模型采用
6. **三代演进** — 正弦(2017, Transformer) → 可学习(2018, GPT-2/BERT) → RoPE(2021, LLaMA/DeepSeek)
7. **实验验证** — 有位置编码时，打乱词序会显著改变注意力模式；无位置编码时几乎无变化

## FAQ

**Q1: 为什么不直接把位置编号（0, 1, 2, ...）作为特征输入？**
因为位置编号是无界的整数，序列越长数字越大，会导致数值不稳定。而且简单的整数无法表达位置之间的复杂关系。正弦编码把位置映射到 [-1, 1] 的有界空间，且不同维度的频率能同时捕获局部和全局的位置关系。

**Q2: 正弦位置编码和可学习位置编码，实际用哪个？**
在原始 Transformer(2017) 论文中，两者效果差不多。GPT-2/BERT 选择了可学习方式（略灵活），但现代大模型已经转向 RoPE，因为它能更好地编码相对位置且支持长度外推。

**Q3: RoPE 为什么比前两种方法更好？**
三个关键优势：(1) 天然编码相对位置——注意力分数只取决于两个词的距离，而非绝对位置；(2) 通过调整频率基数可以外推到训练时未见过的长度；(3) 不增加额外参数，只是在计算注意力前旋转 Q/K 向量。

**Q4: 位置编码是加在输入上还是加在注意力上？**
正弦和可学习位置编码都是加在输入嵌入上（Word Embedding + Positional Encoding）。而 RoPE 不同——它不修改输入，而是在计算注意力时旋转 Q 和 K 向量。这是一个重要的设计差异。

**Q5: 实验中"无 PE"时注意力差异为什么不是精确的 0？**
理论上应该是 0，但浮点运算会引入微小的数值误差（通常在 1e-7 量级）。这不影响结论——与"有 PE"时的显著差异（通常大几百倍）相比，可以忽略不计。
