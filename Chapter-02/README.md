## 🎯 核心知识点

1. **CPU vs GPU**：CPU 拥有少量强大核心（4-16 个），擅长复杂逻辑；GPU 拥有数千个简单核心，擅长并行计算。深度学习的核心是矩阵运算，天然适合 GPU 并行处理
2. **显存（VRAM）管理**：显存是 GPU 专用内存，训练时需要存放模型参数、梯度、优化器状态和中间激活值四大部分，是最紧缺的资源
3. **OOM 应对策略**：遇到显存不足时，按优先级依次尝试——减小 batch size、使用混合精度（FP16）、梯度累积、梯度检查点
4. **Kaggle GPU 配额**：免费提供 T4 GPU（16GB 显存），每周 30 小时，单次 Session 最长 12 小时，需要合理规划使用
5. **CUDA 软件栈**：Python -> PyTorch -> CUDA -> GPU 硬件，`torch.cuda.is_available()` 是验证环境的第一步
6. **FP16 半精度**：在 T4 的 Tensor Core 上可同时提升计算速度（约 2-4 倍）并减半显存占用，是训练优化的基础手段
7. **16GB 显存的能力边界**：可全参数训练约 300M 参数模型，使用 LoRA 可微调 3B-7B 参数模型

## ❓ 常见问题 FAQ

**Q1: `torch.cuda.is_available()` 返回 False 怎么办？**

最常见的原因是 Kaggle Notebook 没有开启 GPU 加速器。点击 Notebook 右侧面板的齿轮图标，找到 Accelerator 选项，选择 GPU T4 x2，保存后重新运行。如果配额已用完（每周 30 小时），则需要等待下周配额重置。

**Q2: 遇到 CUDA Out of Memory (OOM) 错误怎么办？**

按优先级依次尝试：(1) 减小 batch_size（最简单有效），(2) 使用 FP16 半精度训练（显存减半），(3) 使用梯度累积（小 batch 模拟大 batch），(4) 使用梯度检查点（用时间换空间），(5) 换更小的模型。记住口诀：OOM 了？batch 减半！还 OOM？精度减半！还 OOM？模型减半！

**Q3: Kaggle 每周 30 小时 GPU 时间够用吗？**

对于本课程完全够用。关键技巧是：编写和调试代码时使用 CPU 模式（不消耗 GPU 配额），只在实际训练时才开启 GPU。合理规划的话，每周可以有 10+ 小时的余量。千万不要开着 GPU Notebook 然后去休息！

**Q4: 为什么有时候 GPU 计算反而比 CPU 慢？**

当数据量很小时（比如小矩阵运算），CPU 到 GPU 的数据传输开销会抵消 GPU 的计算优势。GPU 的优势体现在大规模并行计算上，矩阵越大、batch 越大，GPU 的加速比越明显。这也是为什么深度学习中尽量使用较大的 batch size。

**Q5: `del tensor` 和 `torch.cuda.empty_cache()` 有什么区别？**

`del tensor` 只是释放 Python 对象的引用，PyTorch 可能仍然保留这块显存作为缓存以备后续使用。`torch.cuda.empty_cache()` 才会真正将未使用的缓存显存释放回系统。遇到 OOM 时，可以先尝试 `torch.cuda.empty_cache()`，但更根本的解决方法还是减小显存占用。
